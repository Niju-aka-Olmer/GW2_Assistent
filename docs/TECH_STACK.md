# GW2 Assistant — Tech Stack

## Выбор технологического стека

### Фронтенд: React 18 + TypeScript + Vite

| Технология | Версия | Зачем |
|-----------|--------|-------|
| **React 18** | ^18.2.0 | Современная компонентная архитектура, хуки, Suspense |
| **TypeScript** | ^5.0.0 | Статическая типизация, автокомплит, предотвращение ошибок |
| **Vite** | ^5.0.0 | Быстрая сборка, HMR (мгновенное обновление), оптимизация |
| **React Router v6** | ^6.20.0 | Клиентская маршрутизация, nested routes, loaders |
| **TanStack React Query** | ^5.0.0 | Кэширование API-запросов, ревалидация, optimistic updates |
| **Tailwind CSS** | ^3.4.0 | Utility-first CSS, тёмная/светлая тема через CSS-переменные |
| **Axios** | ^1.6.0 | HTTP-клиент с интерцепторами, отменой запросов |

### Почему именно этот стек?

1. **React 18** — стандарт индустрии для SPA. Конкурентный рендеринг, автоматическое батчирование, хуки. Для проекта с множеством интерактивных элементов (тултипы, сетки предметов, AI-чат) React даёт предсказуемую производительность.

2. **TypeScript** — критически важен для проекта с интеграцией внешних API (GW2 API возвращает сложные вложенные JSON). Типизация предотвращает ошибки на этапе разработки.

3. **Vite** — современная альтернатива CRA. В 10x быстрее на HMR. Нативная поддержка TypeScript, CSS-модулей, оптимизация бандла.

4. **TanStack React Query** — идеальное решение для кэширования API-запросов. GW2 API имеет rate limits, React Query автоматически кэширует ответы, ревалидирует при необходимости, поддерживает retry и stale time.

5. **Tailwind CSS** — утилитарный CSS даёт гибкость для создания сложной сетки предметов (ItemGrid) и тултипов. Встроенная поддержка тёмной темы через `dark:` префикс. Маленький бандл за счёт purge.

6. **Axios** — перехватчики для единой обработки ошибок, автоматическая сериализация параметров, отмена запросов при уходе со страницы.

### Бэкенд: FastAPI (Python 3.11+)

| Технология | Версия | Зачем |
|-----------|--------|-------|
| **FastAPI** | ^0.104+ | Асинхронный веб-фреймворк, автодокументация OpenAPI |
| **uvicorn** | ^0.24+ | ASGI сервер с горячей перезагрузкой |
| **httpx** | ^0.25+ | Асинхронный HTTP-клиент для внешних API |
| **cachetools** | ^5.3+ | In-memory кэш с TTL |
| **pydantic** | ^2.0+ | Валидация данных, схемы ответов (встроен в FastAPI) |
| **python-dotenv** | ^1.0+ | Переменные окружения |

### Почему FastAPI?

1. **Асинхронность** — бэкенд делает множественные запросы к GW2 API (предметы, цены, персонажи). httpx позволяет выполнять их конкурентно, что даёт x3-5 ускорение.

2. **Автоматическая документация** — Swagger UI (/docs) и ReDoc (/redoc) из коробки. Упрощает отладку интеграции с GW2 API.

3. **Pydantic models** — строгая валидация данных из GW2 API. Если API изменит формат ответа, мы узнаем об этом сразу.

4. **Производительность** — FastAPI сравним с Node.js/Go по скорости, но при этом код остаётся читаемым.

### Сравнение альтернатив

#### Фронтенд

| Критерий | React (выбран) | Vue 3 | Svelte | Vanilla JS |
|----------|---------------|-------|--------|------------|
| Экосистема | Крупнейшая | Большая | Малая | Нет |
| Библиотеки | React Query, Router v6 | Vue Query, Router | Нет | Нет |
| TypeScript | Отличная поддержка | Хорошая | Средняя | Вручную |
| HMR | Vite — мгновенный | Vite — мгновенный | Vite — мгновенный | Вручную |
| Сообщество | Огромное | Большое | Растущее | - |
| Когда выбрать | Сложные SPA с API | Средние проекты | Простые проекты | Очень простые |

**Почему не Vue?** React Query + React Router v6 — зрелые решения для кэширования API и роутинга. У Vue аналогичные библиотеки есть, но React-экосистема для этого проекта даёт больше проверенных решений.

**Почему не Svelte?** Svelte отличен для простых проектов, но для сложной интеграции с API и кэширования экосистема React зрелее.

**Почему не Vanilla JS?** Отсутствие типизации, компонентного подхода, управления состоянием — проект станет неподдерживаемым при добавлении новых страниц.

#### Бэкенд

| Критерий | FastAPI (выбран) | Flask | Node.js (Express) | Django |
|----------|-----------------|-------|-------------------|--------|
| Асинхронность | Нативная | Ограниченная | Нативная | Частичная |
| Скорость | Очень высокая | Средняя | Высокая | Средняя |
| Типизация | Pydantic | Нет | TypeScript | DRF |
| Документация | OpenAPI авто | Вручную | Вручную/swagger | DRF docs |
| Redis-кэш | Легко | Есть | Легко | Есть |

**Почему не Node.js?** FastAPI имеет преимущество в асинхронной работе с несколькими внешними API (GW2 + DeepSeek). httpx позволяет делать параллельные запросы элегантнее, чем axios в Node.js. Кроме того, Python удобнее для работы с текстом (промпты DeepSeek).

**Почему не Flask?** Flask синхронный — при ожидании ответа от GW2 API будет блокироваться. Это приведёт к медленной работе при массовых запросах.

**Почему не Django?** Избыточен для проекта с 5 эндпоинтами. FastAPI легче, быстрее, проще.
